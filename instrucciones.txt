# CHATROOM SERVER - PERSISTENCE LAYER

## Autor:

Persistence layer implementation for distributed systems final project.

## Description:

This module implements the persistence system for the chatroom platform.

Uses JSON files as storage.

The persistence layer was implemented in C using:

* GCC
* cJSON library
* Dynamic memory allocation
* File-based persistence

The module is designed to be integrated later with a multithreaded socket server.

========================================================
FOLDER DESCRIPTION
==================

## include/

Contains all header files (.h).

These files expose:

* Struct definitions
* Public function declarations
* Constants

## src/

Contains implementation files (.c).

These files implement:

* Persistence logic
* JSON parsing
* Memory management
* Index management

## libs/

Contains external libraries.

This project uses:

* cJSON

## data/

Contains persistent storage files.

users.json:
Stores all users.

chatRooms.json:
Stores all chat rooms.

msg.json:
Stores all messages.

index.txt:
Stores next available IDs for entities.

Example:

user: 5
chatroom: 10
message: 25

## Makefile:

Used to compile and execute the project.

========================================================
MODELS
======

## USER

Represents a platform user.

Struct fields:

id
Unique identifier

name
Username

password
User password

chatRoomIds
Dynamic array of chat room IDs

chatRoomCount
Number of chat rooms

## CHATROOM

Represents a chat room.

Struct fields:

id
Unique identifier

name
Chat room name

coordinatorId
User ID of room creator

userIds
Dynamic array of user IDs

userCount
Number of users

messageIds
Dynamic array of message IDs

messageCount
Number of messages

## MESSAGE

Represents a chat message.

Struct fields:

id
Unique identifier

text
Message content

userId
Sender user ID

chatRoomId
Chat room ID

========================================================
PUBLIC FUNCTIONS
================

Only NON-static functions are public and should be used by other modules.

========================================================
USER FUNCTIONS
==============

## createUser

Creates a User struct in memory.

Parameters:

* const char *name
* const char *password

Returns:

* User

## saveUser

Saves a user into users.json.

Parameters:

* User *user

Returns:

* int
  1 = success
  0 = error

## getUserById

Searches a user by ID.

Parameters:

* int id

Returns:

* User *
* NULL if not found

## getAllUsers

Returns all users from users.json.

Parameters:

* int *count

Returns:

* User *

count will contain the number of users.

## updateUser

Updates an existing user in users.json.

Parameters:

* User *user

Returns:

* int

Parameters:

* ChatRoom *

Parameters:

* int *count

* ChatRoom *

Parameters:

* ChatRoom *room

Returns:

* int

* int userId
* int *count

* ChatRoom *

========================================================
MEMORY MANAGEMENT
=================

The system uses dynamic memory allocation.

Functions returning pointers allocate memory dynamically.

The caller is responsible for freeing memory.

Use:

* freeUser()
* freeChatRoom()
* freeMessage()

These functions are implemented in memory_utils.c.

========================================================
JSON LIBRARY (cJSON)
====================

Library source:
https://github.com/DaveGamble/cJSON

Files used:

* cJSON.c
* cJSON.h

The library is included directly in the project.

## Main functions used:

cJSON_Parse
Parses JSON string into cJSON object.

cJSON_Print
Converts cJSON object into JSON string.

cJSON_CreateObject
Creates JSON object.

cJSON_CreateArray
Creates JSON array.

cJSON_AddStringToObject
Adds string field.

cJSON_AddNumberToObject
Adds numeric field.

cJSON_AddItemToArray
Adds item to array.

cJSON_GetObjectItem
Gets object field.

cJSON_GetArrayItem
Gets array element.

cJSON_GetArraySize
Gets array size.

cJSON_Delete
Frees cJSON object.

========================================================
HOW TO MODIFY THE MAKEFILE
==========================

If a new .c file is added:

1. Open Makefile

2. Add file to SRC section.

Example:

SRC = 
src/main.c 
src/database_repository.c 
src/new_file.c 
libs/cJSON.c

Then compile again:

make

========================================================
IMPORTANT NOTES
===============

* Static functions are private to their .c file.
* Only functions declared in headers are public.
* JSON files must always contain valid JSON arrays.
* IDs are managed automatically through index.txt.
* The system is designed to be integrated later with sockets and threads.
* Concurrency control is NOT implemented yet.
* The repository layer is independent from networking logic.

Example architecture:

Socket Server
↓
database_repository
↓
JSON files

========================================================
EXAMPLE USAGE
=============

## Create user:

User user = createUser(
"Carlos",
"1234"
);

saveUser(&user);

## Create chat room:

ChatRoom room = createChatRoom(
"general",
user.id
);

saveChatRoom(&room);

## Create message:

Message msg = createMessage(
"hello world",
user.id,
room.id
);

saveMessage(&msg);

## Get user chat rooms:

int count;

ChatRoom *rooms =
getChatRoomsFromUser(
user.id,
&count
);

## Get room messages:

int count;

Message *messages =
getMessagesFromChatRoom(
room.id,
&count
);
